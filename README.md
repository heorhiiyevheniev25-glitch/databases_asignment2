# databases_asignment2
задача створеного запиту: найпопулярніша та найменш популярна категорії продуктів + середня кількість замовлень по категоріях серед активних клієнтів за 2023 рік

неоптимізований запит:
він не ефективний бо ми три рази виконуємо з'єднання таблиць, фільтрації за умовою та групування.
також ми замість перевірки того чи входить дата в потрібні нам межі ми переводимо кожну дату в значення року та перевіряємо чи не дорівнює воно 2023

оптимізований запит:
використовує cte де ми з'єднуємо таблиці, фільтруємо та групуємо їх. А також тут ми вже використовуємо звичайні межі замість переводу значення для порівняння.
окрім цього я створив індекси на колонках які фільтрують та з'єднують таблиці.

у прикріплених файлах є query плани оптимізованого та неоптимізованого запитів, а також query плани запитів з деякими вимкнутими функціями planner.

також для того що б ви одразу могли дізнатись наскільки прискорився процес після оптимізації я вкажу приблизні значення тут.
